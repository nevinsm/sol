# Survey the Code Area

Broad exploration of the relevant code area identified during orient.

1. Find all files related to the problem area
2. Trace the data flow end-to-end through the relevant paths
3. Read the key functions, types, and interfaces involved
4. Build a map of how the system works in this area
5. Note anything suspicious but do not jump to conclusions

The goal is comprehensive understanding of the code, not a fix. You need a
solid mental model of how this part of the system works before narrowing down.

When you have a thorough understanding of the relevant code, advance:
`sol workflow advance --world=$SOL_WORLD --agent=$SOL_AGENT`
