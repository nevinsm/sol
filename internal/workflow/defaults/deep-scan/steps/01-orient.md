# Orient on the Symptom

Read writ {{issue}} and understand what is being investigated.

1. Read the writ title and description
2. Identify the reported symptom — what is actually happening
3. Identify the expected behavior — what should happen instead
4. Determine which area of the codebase is likely involved
5. Note any reproduction steps, error messages, or logs provided

Do not start exploring code yet. Focus on understanding the problem statement
clearly before moving on.

When you have a clear picture of what is wrong and where to start looking, advance:
`sol workflow advance --world=$SOL_WORLD --agent=$SOL_AGENT`
