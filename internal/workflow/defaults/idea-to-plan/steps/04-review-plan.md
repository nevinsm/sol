# Review the Plan

Consolidate your understanding, requirements, and design into a concrete plan.

Review checklist:
1. Does the plan address the original intent fully?
2. Are the requirements all covered by the design?
3. Are there any gaps — things the plan assumes but does not explicitly handle?
4. Is the ordering correct — can each piece be built and tested independently?
5. Are there any risks or blockers that should be flagged?

Refine the plan if the review surfaces issues. The plan should be a sequence
of discrete, implementable changes — each one a candidate writ.

When the plan passes review, advance:
`sol workflow advance --world={{world}} --agent=$SOL_AGENT`
