# Review Requirements

Study the codebase to understand what exists and what the idea requires.

1. Search for code related to the idea — existing implementations, adjacent
   features, relevant packages
2. Read key files to understand current patterns and conventions
3. Identify what already exists that can be reused or extended
4. List concrete requirements — what must the implementation deliver?
5. Note any constraints: dependencies on other components, compatibility
   concerns, performance considerations

Produce a requirements summary:
- **Must have**: capabilities required for the idea to work
- **Existing**: code or patterns that already support parts of the idea
- **Gaps**: what does not exist yet and must be built

When requirements are clear, advance:
`sol workflow advance --world={{world}} --agent=$SOL_AGENT`
