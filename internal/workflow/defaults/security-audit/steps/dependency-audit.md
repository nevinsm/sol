# Dependency Audit

Audit dependencies for known vulnerabilities, outdated packages, and supply chain risks.

Scope: {{scope}}
{{#focus}}Focus: {{focus}}{{/focus}}

Examine:
- Known CVEs in direct and transitive dependencies
- Outdated packages with available security patches
- Supply chain risks: typosquatting, compromised maintainers, unmaintained packages
- Unnecessary dependencies that expand the attack surface
- Dependency pinning and lock file hygiene
- License compliance issues that may affect security posture
