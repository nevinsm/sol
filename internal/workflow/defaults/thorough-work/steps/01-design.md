# Design

Design the approach for {{issue}} before writing any code.

1. Read the work item title and description carefully
2. Explore the relevant areas of the codebase
3. Identify the files, types, and interfaces that need to change
4. Consider edge cases, failure modes, and testing strategy
5. Write a brief design summary as a comment in your first commit message

Do not start coding yet. The goal is a clear mental model of the change
before touching any files.

When your design is solid, advance:
`sol workflow advance --world=$SOL_WORLD --agent=$SOL_AGENT`
